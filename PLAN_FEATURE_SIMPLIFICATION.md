# 🎯 aisoulcode.cn 功能精简 & UI 优化 — 实施计划

> **目标：** 从64个页面 → 聚焦核心产品，建立商业闭环，提升用户体验
> **风格：** 深色/暖色主题 + 翡翠绿/金色点缀 · 手机优先 · 快

## 一、现状分析

### 当前问题清单

## 二、精简方案

### 战略原则

1. **一个入口走到底**：所有用户只走一条路 → 输入出生信息 → 出融合报告
2. **保留但隐藏**：副线功能不删除代码，通过 `next.config.ts` redirect 做软隐藏
3. **付费墙前置**：基础报告免费，深度内容/AI对话/PDF 下载需付费
4. **移动端优先**：所有交互以手机屏幕为基准设计

### 导航栏精简 → 4 项

| # | 导航项 | 链接 | 说明 |
|---|--------|------|------|
| 1 | ✦ **人生总览** | `/master-report` | 核心产品，首页直接跳转至此 |
| 2 | 🔧 **自助排盘** | `/ziwei` | 老用户专用，直接看各系统原始排盘 |
| 3 | 📖 **命理百科** | `/dharma` | 知识库/博客 |
| 4 | 👤 **关于** | `/about` | 新建的 about 页面（介绍/联系方式） |

> **要隐藏的导航项：** astrology, human-design, fusion, daily, health, hand-diagnosis, compatibility, tcm, tcm-assessment, mbti, payment, admin, tools, my, auth

这些页面**保留代码不删除**，但导航栏不展示，通过 `next.config.ts` 添加 `redirects` 将访问量引导至 master-report。

### 首页改造

**原首页：** 展示多个核心产品卡片 + 列表 → **新首页：** 立即重定向到 `/master-report`

```ts
// next.config.ts 添加
redirects: async () => [
  { source: '/', destination: '/master-report', permanent: true },
  // 其他软隐藏路由
  { source: '/fusion', destination: '/master-report', permanent: false },
  { source: '/astrology', destination: '/master-report', permanent: false },
  { source: '/human-design', destination: '/master-report', permanent: false },
]
```

### 页面保留策略（三档）

| 级别 | 数量 | 页面 | 策略 |
|------|------|------|------|
| **🔵 核心** | 3 | `/master-report`, `/ziwei`（自助排盘）, `/payment` | 深度优化 + 付费墙 |
| **🟢 保留** | 6 | `/dharma`, `/dharma/treasury`, `/dharma/search`, `/tools/matching`, `/tools/orid`, `/about` | 保留功能，导航第二级 |
| **⚪ 软隐藏** | 27+ | 其余所有页面 | 代码保留，路由 redirect 到 master-report，导航不展示 |

## 三、UI 优化方案

### 3.1 首页/人生总览 — 信息输入页优化

**当前问题：** 输入框分散、缺少引导感、移动端适配不佳
**优化方案：**

```
┌─────────────────────────┐
│    ✦ 人生总览             │
│  一次输入·七系统交叉融合    │
│                          │
│  ┌──┬──┬──┬────┐        │
│  │年│月│日│时辰│        │  ← 水平排列，移动端换行
│  └──┴──┴──┴────┘        │
│  [出生地点]              │  ← 渐入式：先填日期，再出现地点
│  [出生时间]              │  ← 自动UTC转换提示
│  (性别选择) 男 / 女      │
│                          │
│  [✦ 生成完整报告 ✦]      │  ← 渐变金绿按钮，全宽
│      免费 · 无需登录      │
└─────────────────────────┘
```

**具体改动：**
1. 输入框使用大号字体（16px+），移动端友好
2. 步骤式引导：Step 1 日期 → Step 2 地点 → Step 3 确认
3. 出生地点改为模糊搜索+热门城市快捷选择
4. 添加"精准 vs 快速"模式切换（精准：全部字段；快速：仅生日）
5. 按钮文案改为"✦ 生成我的生命蓝图"（更有仪式感）

### 3.2 报告展示页优化

**当前问题：** 报告内容以纯 Markdown 为主，图表和文字混排不清晰
**优化方案：**

```
┌─────────────────────────┐
│  ✦ 你的生命蓝图           │
│  1990.06.15 · 北京 · 男  │
│                          │
│  ┌─ 报告导航卡片 ──────┐ │
│  │ 🔮 │ 📈 │ 🎭 │ 💼 │ │  ← 8个维度水平滑动
│  │ 八字 │ 能量 │性格│ 职业│ │
│  └────────────────────┘ │
│                          │
│  === 维度内容 ===        │
│  八字命盘 (SVG图表)      │
│  五行能量 (柱状图)       │
│  AI文字解读 (渐入)       │
│                          │
│  [🔊 语音播放] [📥 PDF] │  ← 付费按钮组
│  [💎 解锁完整AI解读]    │  ← 付费入口
└─────────────────────────┘
```

**具体改动：**
1. 报告顶部添加维度导航滑动条（八字命盘、能量K线、性格解码、职业道路、关系蓝图、财富密码、人生时间线、身心健康）
2. 点击导航锚点滚动到对应章节
3. 每个维度区分"免费预览"和"完整解读"（免费段落短，完整段落需付费）
4. 图表(SVG/echarts) 上方展示，AI 文字下方展示
5. 右上角添加"语音播放"按钮（浏览器TTS，免费）

### 3.3 付费转化设计

**分层模型：**

| 层级 | 价格 | 内容 | 触发时机 |
|------|------|------|----------|
| 🆓 基础报告 | 免费 | 八字盘图 + 五行图表 + 每个维度300字摘要 | 输入信息后立即生成 |
| 💎 深度解读 | ¥9.9/次 | 每个维度完整AI解读(2000+字) + PDF下载 | 点击"解锁完整"按钮 |
| 🌟 年度会员 | ¥199/年 | 全年不限次深度解读 + AI追问 + 历史存档 | 导航"会员"入口 |

**UI 实现：**
1. 报告中的完整段落默认显示前 2 句，其余用**模糊渐变遮罩**
2. 遮罩上放置"💎 解锁完整解读"按钮
3. 点击后弹出支付弹窗（不跳转页面）
4. 支付成功 → 渐入完整内容（使用 `fade-in` 动画）
5. 底部添加"发送到邮箱"（付费功能）

### 3.4 视觉细节优化

| 模块 | 当前 | 优化后 |
|------|------|--------|
| **配色** | 翡翠绿 + 金色点缀 | 保留，增加 10% 对比度确保可读性 |
| **字体** | 默认系统字体 | 添加 `Noto Serif SC`（报告正文）+ `Noto Sans SC`（界面）|
| **卡片圆角** | 16px | 统一为 12px（更现代）|
| **阴影** | 多层阴影 | 简化：hover 时仅提升 + 光照效果 |
| **加载动画** | cosmic-loader | 保留，改为进度条式（展示当前计算阶段）|
| **过渡动画** | fade-in | 增加 slide-in-up (100ms offset per element) |
| **暗色模式** | 跟随系统 | 保留 + 手动切换开关在 Footer |
| **图标** | emoji | 保留 emoji（轻量快速），不引入图标库 |

### 3.5 移动端优先原则

1. **Tap 目标最小 44x44px**（iOS HIG 标准）
2. 输入表单使用 `inputmode` 优化数字键盘
3. 报告导航水平滑动（`overflow-x: auto`）
4. 底部固定操作栏（语音/解锁/分享）
5. 图片/图表懒加载（`loading="lazy"`）

## 六、实施步骤（按顺序执行）

### Step 1: 修改 `Nav.tsx`（~30min）
- 精简 `navLinks` 数组为 4 项：人生总览、自助排盘、命理百科、关于
- 移除 `extraLinks` 数组
- 隐藏用户登录按钮（登录依然可用，但导航不突出展示）
- 移动端汉堡菜单同步更新

### Step 2: 配置路由重定向（~20min）
- 在 `next.config.ts` 添加 `redirects`
- 首页 `/` → `/master-report`
- `/astrology`, `/human-design`, `/fusion`, `/daily`, `/health`, `/hand-diagnosis`, `/compatibility`, `/mbti`, `/mbti/result`, `/tcm`, `/tcm-assessment`, `/tcm-assessment/result`, `/tcm-report`, `/auth/login`, `/auth/register`, `/my`, `/my/cases/[id]`, `/admin` → `/master-report`

### Step 3: 首页改造（~15min）
- `app/page.tsx` 简化为客户端重定向或直接导出 `redirect`

### Step 4: 人生总览页面 UI 优化（~3.5h）
- 输入区域改为步骤式引导
- 报告展示区域添加导航维度滑动条
- 免费/付费内容拆分

### Step 5: 支付流程（~2h）
- 改造 PaywallOverlay 为内嵌弹窗
- 接入支付通知 API
- 支付成功后内容渐入

### Step 6: 创建 /about 页面（~1h）
- 简单页面：介绍灵魂解码、团队（光明喜舍）、联系方式、微信公众号/小红书

### Step 7: 更新 Footer（~20min）
- 删除对隐藏路由的引用
- 只保留：人生总览、自助排盘、命理百科、关于

### Step 8: 全局 UI 微调（~2h）
- 动画优化
- 移动端适配检查
- 暗色模式切换开关
- 字体优化

## 七、风险与缓解

| 风险 | 概率 | 缓解方案 |
|------|------|----------|
| 老用户找不到习惯功能 | 高 | 自助排盘入口保留，redirect 页面提示"此功能已整合到人生总览" |
| 付费转化率低于预期 | 中 | 先上线基础付费墙，收集数据后迭代定价和展示方式 |
| 移动端报告加载慢 | 中 | 图表懒加载、AI 文字流式输出 |
| 破坏现有 SEO | 中 | 所有 redirect 使用 302（临时）而非 301，观察流量变化后再决定是否永久 |
| 中医/法藏用户流失 | 低 | 这些用户占比极小，且法藏知识库仍可通过 "/dharma" 访问 |
